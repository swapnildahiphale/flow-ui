Tasks flow-ui in flight

Build a local web UI to visualize flow tasks, projects, and KB

flow-ui · flow-itself · session live · started 16:34

Brief

What

Local web UI that surfaces everything in ~/.flow — tasks, projects, KB, updates, tags, waiting_on, sessions — plus visualizations and insights derived from that data.

Why

  • Faster status scanning than CLI list/show output.
  • Visualize the knowledge graph: KB ↔ tasks ↔ projects ↔ people.
  • Surface insights the CLI can't easily show.
  • A playground for what else flow's data can tell us.

Done when

  • Reads ~/.flow/flow.db directly (read-only).
  • Tasks/projects browser with filters.
  • KB viewer with cross-references.
  • Knowledge graph visualization (Cytoscape).

Pre-flight matrix
  • ✓ Geist + Geist Mono only — Inter banned
  • ✓ Single accent (emerald). Saturation under 80%. No purple-blue gradients.
  • ✓ Cards only where elevation has meaning. `divide-y` for sparse data.
  • ✓ No 3-equal-cards. Asymmetric or zigzag.
  • ✓ Mono `tabular-nums` for IDs and timestamps.
  • ✓ `min-h-[100dvh]`, never `h-screen`.
  • ✓ Mobile: collapses to single column below `md:`.
  • ✓ Phosphor regular-weight icons. No emojis anywhere.
  • ✓ Empty, loading, error states implemented per page.
  • ✓ Perpetual motion isolated + memoized in leaf components.
